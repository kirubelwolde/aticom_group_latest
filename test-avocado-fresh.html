Avocado Fresh Integration Test

Test Instructions

This page helps you test the integration between the admin page and the public page for Avocado Fresh content.

Step 1: Test Admin Page

1.1: Open the admin page: http://localhost:8080/admin/avocado-fresh

1.2: Verify that the page loads and shows the BusinessContentAdmin interface

1.3: Make some changes to the content (e.g., update the page title, hero section, etc.)

1.4: Click the "Save Changes" button

1.5: Verify that you see a success message

Step 2: Test Public Page

2.1: Open the public page: http://localhost:8080/avocado-fresh

2.2: Verify that the page loads and displays the dynamic content

2.3: Check if the changes you made in the admin page are reflected here

2.4: Verify that the page shows the DynamicBusinessPage content instead of hardcoded content

Step 3: Verify Integration

3.1: Go back to the admin page and make another change

3.2: Save the changes

3.3: Refresh the public page and verify the new changes appear

Expected Results

  • Admin page should load and allow editing of content
  • Changes should save successfully
  • Public page should display dynamic content from the database
  • Changes made in admin should immediately reflect on the public page
  • No hardcoded content should be visible on the public page

Common Issues

  • Database connection issues: Make sure Supabase is running and configured
  • Content not loading: Check browser console for errors
  • Changes not saving: Verify RLS policies and database permissions
  • Hardcoded content still showing: Make sure the AvocadoFresh.tsx file was updated correctly
